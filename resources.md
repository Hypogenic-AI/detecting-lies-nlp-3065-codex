### Resource Gathering Notes

#### Search Strategy
- Primary: `paper-finder` diligent mode with query focused on lie detection, hallucination, confabulation, and strategic deception.
- Secondary: Semantic Scholar metadata resolution + arXiv title fallback for open-access PDFs.
- Dataset search: Hugging Face Hub API + `datasets` downloads with config-aware retries.
- Code search: extracted GitHub links from downloaded PDFs and cloned relevant benchmark repos.

#### Selection Criteria
- Prioritized papers that explicitly separate deception from hallucination or study lie detection mechanisms.
- Included datasets with direct utility for epistemic-failure vs incentive-misreporting experiments.
- Chosen repositories provide executable baselines/benchmarks for deception evaluation.

#### Challenges Encountered
- Many Semantic Scholar results did not expose open-access PDFs directly.
- One downloaded PDF was corrupted and removed.
- Some Hugging Face datasets required explicit config names or were script-only in this environment.

#### Gaps and Workarounds
- For unavailable PDFs, used arXiv fallback search by exact title.
- For incompatible datasets, substituted with accessible sycophancy/deception variants.

### Recommendations for Experiment Design

1. **Primary dataset(s)**: TruthfulQA (epistemic), HaluEval (hallucination), DeceptionBench/Deception-evals + sycophancy datasets (incentive-driven misreporting).
2. **Baseline methods**: activation-based lie detectors (truth direction), uncertainty-based hallucination detectors, prompting/steering honesty baselines.
3. **Evaluation metrics**: accuracy/F1 on truthful vs false outputs, deception rate under pressure/reward conditions, calibration/abstention metrics, and trust-decay metrics in multi-turn settings.
4. **Code to adapt/reuse**: `DeceptionBench`, `LongHorizonDeception`, `alignment-for-honesty`, and `hoodwinked` for scenario generation/evaluation.
